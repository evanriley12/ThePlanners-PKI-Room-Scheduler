 {\ttfamily to\+Throw()} (and its alias {\ttfamily to\+Throw\+Error()}) is used to check if an error is thrown by a function call, such as in {\ttfamily expect(() =\texorpdfstring{$>$}{>} a()).to\+Throw()}. However, if no message is defined, then the test will pass for any thrown error. Requiring a message ensures that the intended error is thrown.\hypertarget{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2eslint-plugin-jest_2docs_2rules_2require-to-throw-message_autotoc_md10486}{}\doxysection{\texorpdfstring{Rule details}{Rule details}}\label{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2eslint-plugin-jest_2docs_2rules_2require-to-throw-message_autotoc_md10486}
This rule triggers a warning if {\ttfamily to\+Throw()} or {\ttfamily to\+Throw\+Error()} is used without an error message.\hypertarget{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2eslint-plugin-jest_2docs_2rules_2require-to-throw-message_autotoc_md10487}{}\doxysubsection{\texorpdfstring{Default configuration}{Default configuration}}\label{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2eslint-plugin-jest_2docs_2rules_2require-to-throw-message_autotoc_md10487}
