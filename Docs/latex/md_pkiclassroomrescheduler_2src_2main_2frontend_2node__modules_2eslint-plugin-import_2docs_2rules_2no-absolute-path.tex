Node.\+js allows the import of modules using an absolute path such as {\ttfamily /home/xyz/file.js}. That is a bad practice as it ties the code using it to your computer, and therefore makes it unusable in packages distributed on {\ttfamily npm} for instance.

By default, only ES6 imports and Common\+JS {\ttfamily require} calls will have this rule enforced.

You may provide an options object providing true/false for any of


\begin{DoxyItemize}
\item {\ttfamily esmodule}\+: defaults to {\ttfamily true}
\item {\ttfamily commonjs}\+: defaults to {\ttfamily true}
\item {\ttfamily amd}\+: defaults to {\ttfamily false}
\end{DoxyItemize}

If {\ttfamily \{ amd\+: true \}} is provided, dependency paths for AMD-\/style {\ttfamily define} and {\ttfamily require} calls will be resolved\+:
