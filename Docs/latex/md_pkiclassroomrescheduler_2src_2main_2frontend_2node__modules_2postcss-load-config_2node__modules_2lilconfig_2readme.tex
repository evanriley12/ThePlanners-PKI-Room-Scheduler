Specifically {\ttfamily js} files in projects with {\ttfamily "{}type"{}\+: "{}module"{}} in {\ttfamily package.\+json} or {\ttfamily mjs} files.\hypertarget{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2postcss-load-config_2node__modules_2lilconfig_2readme_autotoc_md18884}{}\doxysection{\texorpdfstring{Difference to {\ttfamily cosmiconfig}}{Difference to {\ttfamily cosmiconfig}}}\label{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2postcss-load-config_2node__modules_2lilconfig_2readme_autotoc_md18884}
Lilconfig does not intend to be 100\% compatible with {\ttfamily cosmiconfig} but tries to mimic it where possible. The key difference is {\bfseries{no}} support for yaml files out of the box({\ttfamily lilconfig} attempts to parse files with no extension as JSON instead of YAML). You can still add the support for YAML files by providing a loader, see an example below.\hypertarget{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2postcss-load-config_2node__modules_2lilconfig_2readme_autotoc_md18885}{}\doxysubsection{\texorpdfstring{Options difference between the two.}{Options difference between the two.}}\label{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2postcss-load-config_2node__modules_2lilconfig_2readme_autotoc_md18885}
