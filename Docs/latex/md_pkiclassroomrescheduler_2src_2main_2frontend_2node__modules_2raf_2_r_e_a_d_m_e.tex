{\ttfamily callback} is the function to invoke in the next frame. {\ttfamily handle} is a long integer value that uniquely identifies the entry in the callback list. This is a non-\/zero value, but you may not make any other assumptions about its value.\hypertarget{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2raf_2_r_e_a_d_m_e_autotoc_md20199}{}\doxysubsection{\texorpdfstring{raf.\+cancel(handle)}{raf.\+cancel(handle)}}\label{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2raf_2_r_e_a_d_m_e_autotoc_md20199}
{\ttfamily handle} is the entry identifier returned by {\ttfamily raf()}. Removes the queued animation frame callback (other queued callbacks will still be invoked unless cancelled).\hypertarget{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2raf_2_r_e_a_d_m_e_autotoc_md20200}{}\doxysubsection{\texorpdfstring{raf.\+polyfill(\mbox{[}object\mbox{]})}{raf.\+polyfill(\mbox{[}object\mbox{]})}}\label{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2raf_2_r_e_a_d_m_e_autotoc_md20200}
Shorthand to polyfill {\ttfamily window.\+request\+Animation\+Frame} and {\ttfamily window.\+cancel\+Animation\+Frame} if necessary (Polyfills {\ttfamily global} in node).

Alternatively you can require {\ttfamily raf/polyfill} which will act the same as `require(\textquotesingle{}raf').polyfill()\`{}.

If you provide {\ttfamily object} the polyfills are attached to that given object, instead of the inferred global.
