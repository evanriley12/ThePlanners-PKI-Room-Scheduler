Convert bytes to a human readable string\+: {\ttfamily 1337} → {\ttfamily 1.\+34 kB} \end{quote}

Type\+: {\ttfamily boolean}\textbackslash{} Default\+: {\ttfamily false}

Include plus sign for positive numbers. If the difference is exactly zero a space character will be prepended instead for better alignment.\hypertarget{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2pretty-bytes_2readme_autotoc_md19746}{}\doxyparagraph{\texorpdfstring{bits}{bits}}\label{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2pretty-bytes_2readme_autotoc_md19746}
Type\+: {\ttfamily boolean}\textbackslash{} Default\+: {\ttfamily false}

Format the number as \href{https://en.wikipedia.org/wiki/Bit}{\texttt{ bits}} instead of \href{https://en.wikipedia.org/wiki/Byte}{\texttt{ bytes}}. This can be useful when, for example, referring to \href{https://en.wikipedia.org/wiki/Bit_rate}{\texttt{ bit rate}}.\hypertarget{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2pretty-bytes_2readme_autotoc_md19747}{}\doxyparagraph{\texorpdfstring{binary}{binary}}\label{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2pretty-bytes_2readme_autotoc_md19747}
Type\+: {\ttfamily boolean}\textbackslash{} Default\+: {\ttfamily false}

Format the number using the \href{https://en.wikipedia.org/wiki/Binary_prefix}{\texttt{ Binary Prefix}} instead of the \href{https://en.wikipedia.org/wiki/SI_prefix}{\texttt{ SI Prefix}}. This can be useful for presenting memory amounts. However, this should not be used for presenting file sizes.\hypertarget{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2pretty-bytes_2readme_autotoc_md19748}{}\doxyparagraph{\texorpdfstring{locale}{locale}}\label{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2pretty-bytes_2readme_autotoc_md19748}
Type\+: {\ttfamily boolean \texorpdfstring{$\vert$}{|} string}\textbackslash{} Default\+: {\ttfamily false} \texorpdfstring{$\ast$}{*}(No localization)\texorpdfstring{$\ast$}{*}

{\bfseries{Important\+:}} Only the number and decimal separator are localized. The unit title is not and will not be localized.


\begin{DoxyItemize}
\item If {\ttfamily true}\+: Localize the output using the system/browser locale.
\item If {\ttfamily string}\+: Expects a \href{https://en.wikipedia.org/wiki/IETF_language_tag}{\texttt{ BCP 47 language tag}} (For example\+: {\ttfamily en}, {\ttfamily de}, …)
\item If {\ttfamily string\mbox{[}\mbox{]}}\+: Expects a list of \href{https://en.wikipedia.org/wiki/IETF_language_tag}{\texttt{ BCP 47 language tags}} (For example\+: {\ttfamily en}, {\ttfamily de}, …)
\end{DoxyItemize}

{\bfseries{Note\+:}} Localization should generally work in browsers. Node.\+js needs to be \href{https://github.com/nodejs/node/wiki/Intl}{\texttt{ built}} with {\ttfamily full-\/icu} or {\ttfamily system-\/icu}. Alternatively, the \href{https://github.com/unicode-org/full-icu-npm}{\texttt{ {\ttfamily full-\/icu}}} module can be used to provide support at runtime. \href{https://nodejs.org/en/blog/release/v13.0.0/}{\texttt{ Node.\+js 13}} and later ships with ICU by default.\hypertarget{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2pretty-bytes_2readme_autotoc_md19749}{}\doxyparagraph{\texorpdfstring{minimum\+Fraction\+Digits}{minimum\+Fraction\+Digits}}\label{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2pretty-bytes_2readme_autotoc_md19749}
