To begin, you\textquotesingle{}ll need to install {\ttfamily file-\/loader}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\$\ npm\ install\ file-\/loader\ -\/-\/save-\/dev}

\end{DoxyCode}


Import (or {\ttfamily require}) the target file(s) in one of the bundle\textquotesingle{}s files\+:

\begin{DoxyCode}{0}
\DoxyCodeLine{import\ img\ from\ './file.png';}

\end{DoxyCode}

An application might want to configure different CDN hosts depending on an environment variable that is only available when running the application. This can be an advantage, as only one build of the application is necessary, which behaves differently depending on environment variables of the deployment environment. Since file-\/loader is applied when compiling the application, and not when running it, the environment variable cannot be used in the file-\/loader configuration. A way around this is setting the {\ttfamily \+\_\+\+\_\+webpack\+\_\+public\+\_\+path\+\_\+\+\_\+} to the desired CDN host depending on the environment variable at the entrypoint of the application. The option {\ttfamily post\+Transform\+Public\+Path} can be used to configure a custom path depending on a variable like {\ttfamily \+\_\+\+\_\+webpack\+\_\+public\+\_\+path\+\_\+\+\_\+}.
