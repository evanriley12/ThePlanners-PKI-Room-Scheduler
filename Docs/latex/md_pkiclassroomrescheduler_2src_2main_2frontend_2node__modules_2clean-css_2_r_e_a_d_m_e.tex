clean-\/css 5.\+0 introduced some breaking changes\+:


\begin{DoxyItemize}
\item Node.\+js 6.\+x and 8.\+x are officially no longer supported;
\item {\ttfamily transform} callback in level-\/1 optimizations is removed in favor of new plugins interface;
\item changes default Internet Explorer compatibility from 10+ to \texorpdfstring{$>$}{>}11, to revert the old default use `\{ compatibility\+: \textquotesingle{}ie10' \}{\ttfamily flag;}
\item {\ttfamily changes default}rebase{\ttfamily option from}true{\ttfamily to}false{\ttfamily so URLs are not rebased by default. Please note that if you set}rebase\+To{\ttfamily option it still counts as setting}rebase\+: true\`{} to preserve some of the backward compatibility.
\end{DoxyItemize}

And on the new features side of things\+:


\begin{DoxyItemize}
\item format options now accepts numerical values for all breaks, which will allow you to have more control over output formatting, e.\+g. {\ttfamily format\+: \{breaks\+: \{after\+Comment\+: 2\}\}} means clean-\/css will add two line breaks after each comment
\item a new {\ttfamily batch} option (defaults to {\ttfamily false}) is added, when set to {\ttfamily true} it will process all inputs, given either as an array or a hash, without concatenating them.
\end{DoxyItemize}\hypertarget{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2clean-css_2_r_e_a_d_m_e_autotoc_md6903}{}\doxysubsection{\texorpdfstring{What\textquotesingle{}s new in version 4.\+2}{What\textquotesingle{}s new in version 4.\+2}}\label{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2clean-css_2_r_e_a_d_m_e_autotoc_md6903}
