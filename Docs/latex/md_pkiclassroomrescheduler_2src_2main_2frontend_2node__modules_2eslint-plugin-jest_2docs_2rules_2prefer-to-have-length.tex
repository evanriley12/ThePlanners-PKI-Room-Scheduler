 In order to have a better failure message, {\ttfamily to\+Have\+Length()} should be used upon asserting expectations on objects length property.\hypertarget{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-to-have-length_autotoc_md10477}{}\doxysection{\texorpdfstring{Rule details}{Rule details}}\label{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-to-have-length_autotoc_md10477}
This rule triggers a warning if {\ttfamily to\+Be()}, {\ttfamily to\+Equal()} or {\ttfamily to\+Strict\+Equal()} is used to assert objects length property.


\begin{DoxyCode}{0}
\DoxyCodeLine{expect(files.length).toBe(1);}

\end{DoxyCode}


This rule is enabled by default.\hypertarget{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-to-have-length_autotoc_md10478}{}\doxysubsection{\texorpdfstring{Default configuration}{Default configuration}}\label{md_pkiclassroomrescheduler_2src_2main_2frontend_2node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-to-have-length_autotoc_md10478}
The following patterns are considered warnings\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{expect(files.length).toBe(1);}
\DoxyCodeLine{}
\DoxyCodeLine{expect(files.length).toEqual(1);}
\DoxyCodeLine{}
\DoxyCodeLine{expect(files.length).toStrictEqual(1);}

\end{DoxyCode}


The following pattern is not warning\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{expect(files).toHaveLength(1);}

\end{DoxyCode}
